Importing data from external sources

When working with table data in 1C:Drive (in lists, in documents, and so on), you may need to enter information that is originally stored in electronic spreadsheets like Microsoft Excel files.

You can manually retype the data into a table from such a spreadsheet or copy and paste the data cell by cell.

Instead, you may prefer to import the data of the entire spreadsheet or its selected records to the matching columns of the table you are populating in 1C:Drive. This significantly reduces the time and effort you spend on data entry.

You simply either select the spreadsheet file whose data you want to import, or paste the spreadsheet content from the clipboard, then set the mapping between the columns of the spreadsheet and the columns of the target table, mark the records you are going to import, and finally have the data imported automatically.

For instance, you can use this capability to populate the list of customers and the list of products.

You can import data from XLSX, MXL, or CSV files.

To import data from a spreadsheet stored outside of 1C:Drive to a table in 1C:Drive:

  1. Open the table.
  2. Click Import data from an external source .
  3. Do any of the following:
    • To import all data from the spreadsheet file, click import data from external file and select the file.
    • To import selected data from the spreadsheet, open the spreadsheet in an application that supports the spreadsheet's format (like Microsoft Excel), copy the spreadsheet content to the clipboard, then, in 1C:Drive, click copy and paste data to spreadsheet template, and paste the data from the clipboard.

    1C:Drive displays the content of the spreadsheet in the template.

    Here you can edit the data in cells and also insert and delete rows likewise you do this in Microsoft Excel.

    By default, the header of each column in the template contains the "Do not import" text, which means that you are not going to import the column's data to the table.

  4. For each column you want to import to a table column, do the following:
    1. Click the column header.

      1C:Drive displays the list of available table columns.

      You have to provide the mapping at least for required table columns, which are marked in red. Table columns that you have already used in the mapping are marked in gray.

    2. Double-click the name of the target table column.

      Now 1C:Drive displays the name of the target table column in the header of the column in the template.

    3. If you have already mapped a column to a target table column and you want to clear the mapping, click the column's header and double-click Do not import.
  5. Click Next.
  6. Do any of the following:
    • If there are errors or warnings, review them, click Back to return to the previous step, correct the mapping, and then click Next twice to proceed to the next step.
    • If there are no errors and warnings at all, or there are warnings you can treat as insignificant ones, click Next to proceed to the next step.

    1C:Drive lets you review the data to be imported and the data in the target table, compare them record by record, and decide what to do with matched and unmatched items.

    A matched item is an external spreadsheet row for which 1C:Drive found the matching row in its table. An unmatched item is an external spreadsheet row for which there is no matching row in the table.

    To review the matches, click Show rows and then do any of the following:

    • To view all items, in the Show in table list, select All data.
    • To view only matched items, in the Show in table list, select Matched.
    • To view only unmatched items, in the Show in table list, select Data with no match in the database.
    • To view only items that cannot be imported because they are incomplete or contain errors, in the Show in table list, select Data containing error (incomplete).

    To add data from the matched template rows to the table, select the Update mapped items check box.

    To skip adding data from the matched template rows to the table, clear the Update mapped items check box.

    To add data from the unmatched template rows to the table as new rows, select the Create new items if not mapped check box.

    To skip adding data from the unmatched template rows to the table, clear the Create new items if not mapped check box.

    To specify the group to which you want to add new rows, in the Group to import in field, select the target group.

  7. Click Import data to application.

Now you can verify the imported data in the resulting table.
