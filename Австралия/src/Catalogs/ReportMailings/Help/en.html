Use this list to configure rules for automatic report generation and delivery.

You can open the list by clicking Report distribution rule sets in Service subsystems section.

The application features a variety of reports. Users often require the same report periodically. For example, a manager might need a daily report on mutual payments state. A CEO might need a weekly report on a company's key financial indicators.

Report distribution rule sets automate this process. Users get the reports they need by email or from a shared folder, without the need to run the application.

You can create multiple report distribution rule sets. A rule set usually groups several report types or recipients, or features specific periodicity.

For example, a report distribution rule set can include reports intended solely for a CEO and top managers. Another rule set can contain daily reports that are sent to some employees. A third rule set can contain reports sent to counterparties upon their request.

Once you configure a rule set, you can schedule the report distribution using the "Report distribution" scheduled job, or send them manually.

The list has the following columns:

  • Description. The full name of a rule set or a group of rule sets.
  • Prepared. Shows whether the rule set is ready to use. If this check box is cleared, the rule set never generates or distributes reports.
  • Last launch. The date and time of the last report distribution. If distribution failed, this column displays the following icon: .
  • Schedule. A text presentation of the report distribution schedule.
  • You can click More actions—Change form to add more columns:
    • Author. The user on whose behalf scheduled reports are sent.
    • State. The result of the last report distribution attempt: Completed or Not completed.
    • Archive. This check box is selected if the Compress to ZIP archive check box is selected for the rule set.
    • Send to. The list that contains the report recipients.
    • Address type. If the rule set includes sending reports by email, this field contains Email.

Adding report distribution rule sets

  • Click Create and fill in the fields.

Grouping report distribution rule sets

  • Click Create group and fill in the fields.
  • Drag the rule sets to groups.

Running report distribution

  • To run report distribution based on the selected rule sets and groups of rule sets on behalf of the current user, сlick Run now.

Viewing report distribution events

  • To view report distribution errors, click Report distribution events. This opens event log events filtered by report distribution rule set.
