Use this window to generate reports automatically and deliver the reports to their recipients.

Report distribution rule sets automate report distribution. Users get the reports they need by email or from a shared folder, without the need to run the application.

General parameters

  • Description. A rule set name as it appears in lists. You can specify it as a template.
  • Prepared. Shows whether the rule set is ready to use. When you select this check box, some of the fields become required.
  • Send. A report distribution type, which depends on the purpose:
    • Reports to the specified recipients. Send identical reports to all recipients. In addition to sending by email, this report distribution type supports publishing generated reports to an application files folder, a network folder, or an FTP server.
      • In the Recipients field, select the recipient type. For example, Users.
    • Unique report for each recipient. Personalize a report for each recipient. If you select this distribution type, the report filter settings will include a recipient field, providing the option to customize a report for each recipient.
      • In the Recipients field, select the recipient type.
    • To me only (private bulk email of the author). Send reports by email only to the user who created the rule set. Private report distribution rule sets are visible only to their authors.

      • Email. This field is filled automatically with the email address of the current user.

Schedule

  • Use this tab to configure automatic report distribution. You can set a report distribution period explicitly or specify days of the week, dat, and time for sending reports. Each report distribution rule set generates "Report distribution" scheduled jobs with names mathching the rule set names.
  • Select the Execute on schedule check box.
  • To generate a schedule automatically, click Fill in from template. Select a schedule from the list, for example, On the first day of the month. Then the schedule will be filled automatically.
  • You can select a report distribution period (this will change all of the other schedule parameters). The following periods are available:
    • Daily. Send reports with the period specified in days.
      • Select a Period.
    • Weekly. Send reports on specific days of the week.
      • Select days of the week.
    • Monthly. Send reports monthly on the specified day of the month.
      • Select months and specify the number of the day from the beginning or end of the month.
    • Custom. Set up a detailed schedule. Selecting this option opens the Schedule window where you can specify a detailed schedule. You can edit a custom schedule later. Once you close the window, the Change button becomes available. Click it to edit the schedule. In this scenario, specify the Start time in the Schedule window.
  • Specify the report distribution Start time.
  • In the Author field, select a user. By default, this field contains the user that created the rule set.
    • Scheduled report distributions run on behalf of their authors.

Distribution

  • Use this tab to specify report distribution options. The set of fields on this tab depends on the distribution type.
  • For private bulk email of the author, reports are sent to the email address specified in the Email field.
  • For Unique report for each recipient, reports are sent by email to multiple recipients.
  • For Reports to the specified recipients, reports can be sent by email or saved (published). To have reports published, select the Publish check box. Select one of the following:
    • To directory. Save reports to the specified directory of the "File operations" subsystem. You can use this option to archive reports to a folder (and turn off sending reports by email).
    • To network directory. Save reports to the specified network directory. Since reports are copied from the server, it is recommended that you specify the network path in UNC format, for example: \\server\folder\. Enter the address in the Windows or Linux field (depending on your operating system). Warning! This report delivery option is not available in web applications.
    • To FTP resource. Save reports to an FTP server. Fill in the required parameters.
  • Check. Test publishing (save and delete a test file). If no errors occur during the test, publishing is considered successful.
  • Send by email. Indicates whether reports or notifications are sent by email.
  • Select recipients. Follow the link to fill in the recipients list and exceptions from the list. You can select recipients or recipient groups. After you fill the list, the link text changes to the first recipient in the list.
  • В поле От из выпадающего списка выберите учетную запись для отправки. This field is required. The list contains only available accounts. You cannot create a new one here.
  • Check. Test email account (send a test email). If no errors occur during the test, publishing is considered successful.
  • If you turn on both publishing and sending by email, you can choose to Send notifications only. In this scenario email messages contain only notifications that reports were generated. Reports are not sent as attachments but stored to the publishing directory. Email notifications contain links to report directories.
  • If you select the Send notifications only check box, the email subject and body fields become available. They can include templates of report distribution parameters. The fields are filled with the default templates. To add parameters to the email, click Parameter and select one of the following:
    • [ExecutionDate]. A report distribution date. You can add this parameter to an archive name.
    • [Author]. A user that initiated report distribution.
    • [Recipient]. A recipient presentation. It is only available for Unique report for each recipient distribution type.
    • [MailingDescription]. A name of a report distribution rule set. You can add this parameter to an archive name.
    • [CreatedReports]. The list of reports and their formats.
    • [SystemTitle]. The infobase title. You can specify it in section Administration—General settings.
    • [TransportMethod]. The presentation of the report publishing resource.
  • The default email format is HTML. You can change it to Plain text. In the HTML format, standard text formatting options are available as buttons or items of the More actions menu.
  • If you select the Transliterate file names check box, The names of report files will contain only Latin letters and digits. This ensures compatibility between different operating systems. For example, file Отчет.mxl will be saved as Otchet.mxl.

More

  • The Default report formats field stores default report formats. Select the check boxes for formats that are common for all reports. You can select multiple formats.
  • In the table below, in the Formats column, you can select an individual set of formats for each report.
  • In the Send blank reports column, select the check boxes for reports that you want sent even if they are blank. By default, the check box is cleared. Reports that contain pictures are never considered blank.
  • Select the Compress to ZIP archive check box to archive generated reports in .zip format. It is recommended that you enable archiving if reports are sent by email to reduce the size of attachments.
    • The Archive description field becomes available (an archive description is generated automatically from a template specified in the field).
    • You can change a description template by right-clicking Parameter in the Archive description field. The following parameters can be changed:
      • Report distribution date
      • Name of a report distribution rule set.
    • If necessary, write an archive Password.
  • In the Reply to field, specify an email address to which report distribution responses will be sent.
  • Select the Send emails using BCC check box so that recipients do not know who else gets reports from this bulk email.
  • Group. The report distribution rule sets can be organized by a variety of criteria. To place a report distribution rule set in a group, select it from the list.

Distributing reports

  • Click Execute now to distribute reports as the current user. The rule set will be sent if it has the Prepared check box selected.

Viewing report distribution events

  • Нажмите События рассылки, для того чтобы открыть журнал регистрации, предварительно отфильтровав его по событиям рассылки со времени последнего запуска.

Analyzing report distribution results

To control report distribution results, you can use:

  • Sent folder control. Sent emails are usually put in this folder on the email server. If it is not so, check the email server settings and fix them.
  • Additional addresses for report distribution If emails do not go to the Sent folder or further control is required, you can add your own mailbox to the list of report distribution addresses. You can select the Send email BCC to this address check box in the email account card (other recipients will not see email copies; the check box is available for POP email servers only) and set up a rule for sorting incoming emails to ensure that such emails go to the correct folder.

Работа в клиент-серверном режиме

  • When running in client/server mode, you should note that bulk email is performed on the 1C:Enterprise server.
  • A path to one resource may be different for client and server depending on their network location and operating systems.
  • If 1C:Enterprise server runs on Windows, you should make sure that the operating system user on whose behalf the 1C:Enterprise server runs has sufficient rights to read and write in the Windows network directory. If the user does not have the sufficient rights, you will need to grant them these rights, or select another user with sufficient rights to read and write in the Windows network directory and run the 1C:Enterprise server on their behalf. The same recommendations apply to FTP and email access.
  • The Linux network directory field is for 1C:Enterprise servers running on Linux. In this field, you can specify only a local path, for example, /home/path.
  • Before you can use UNC path, you need to mount the folder. Sign in as administrator (root) and execute the following commands:
    • mkdir folder
       . Write a directory name instead of folder. For example, execute the mkdir shared command in the /home directory to create the shared directory with the /home/shared full path.
    • mount -t cifs //server/share folder -o //domain/user, password=pwd
      , where //server/share is the UNC path to other computer, folder is the directory path on Linux computer, for example \home\shared, //domain/user is a user/domain name, if needed, and pwd is a user password (if
      needed).
    • If user name and password are not required, simply execute
      mount -t cifs //server/share folder
    • Now you can specify folder (example: /home/shared) as the path to the volume.
  • For more information on 1C:Enterprise server administration, see the 1C:Enterprise platform documentation.
