Details of performer role properties.

Performer roles can be used in business processes for role-based task routing purposes. In this case, tasks are initially assigned to roles (example: Room supervisor) rather than to a specific performer. Map of users and roles are defined in the Roles and task performers list, on which basis the tasks are automatically reassigned to specific users.

However, in certain cases, the performer role is not enough to accurately identify the performers. In these cases, you need to specify the main and additional addressing objects. For example, the Project manager role can have the main Department addressing object and additional Project object. Thus, addressing objects help to clarify to which project managers and in which department the task is assigned.

The list of roles is included in the application, but can be expanded if necessary.

Info about performer role

  • Write the full role description in the Description field.
  • Code. Filled automatically.
  • Write short role description for your convenience in the Short presentation field. By default, Code is automatically copied into this field.
  • In the Assignment field click the reference, using the checkboxes, select the type of performers for whom this role is intended (from the enumerated application lists). Click Select. After that, the reference enumerates the selected values.

Addressing tasks to performers

  • Select the matching check box if Role can be assigned to performers without specifying clarifying objects.
  • To select addressing objects, select the Role is assigned to performers with required specification check box.
    • In the Main addressing objects field, select the All addressing objects value from the list or a specific task addressing object.
    • The Additional addressing object field is filled in the same way.
  • Write additional info about the performer role in the Comment field.

Peculiarities of interaction with external users

  • External users can be performers of business processes (for example, partners or contact persons of partners).
  • As a rule, external users cannot use the names of specific employees of a company, thus, they can only specify the performer roles assigned to them.
  • To do that, you need to configure a list of such roles, specifying that they are designed for certain categories of external users.

Role performers

  • To view the Role performers list, and to assign performers for this role, execute the matching command in the navigation panel.
  • Or you can double-click the role. Several performers can be assigned to the role.
