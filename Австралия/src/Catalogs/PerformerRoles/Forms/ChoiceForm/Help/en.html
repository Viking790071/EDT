Designed for performer role selection.

Opened by clicking Select from the Performer role field of business processes and the window of performer roles selection.

Роли исполнителей могут использоваться бизнес-процессами для ролевой маршрутизации задач. In this case, tasks are initially assigned to roles (example: Room supervisor) rather than to a specific performer. Map of users and roles are defined in the Roles and task performers list, on which basis the tasks are automatically reassigned to specific users. However, in certain cases, the performer role is not enough to accurately identify the performers. In these cases, you need to specify the main and additional addressing objects. For example, the Project manager role can have the main Department addressing object and additional Project object. Thus, addressing objects help to clarify to which project managers and in which department the task is assigned.

The map of performers to roles and addressing object is set before starting work with business processes and tasks, and it is used by the application for delivering instructions to specific performers.

The list of roles is included in the application, but can be expanded if necessary.

Roles with no performers assigned are colored in red. It is not recommended to allow such situations as it can lead to the emergence of tasks that are not completed on time. The Hung tasks report is designed to control such tasks.

The list includes the following columns:

  • Description. Full role description.
  • Short presentation. Short description of a role for your convenience. By default, Code is automatically copied into this field.
  • Code. Filled automatically.
  • Comment. Role details are displayed at the bottom of the list.

Performer role choice

  • Select a role and click Select.
  • You can also use a double-click to select a role.

Creating new role

  • Select Create and fill in the required fields.
