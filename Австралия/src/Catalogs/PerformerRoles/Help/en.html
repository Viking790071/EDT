Designed to manage performer roles.

Opened by the Performer roles command in the Business processes and tasks section.

Performer roles can be used in business processes for role-based task routing purposes. In this case, tasks are initially assigned to roles (example, Room supervisor) rather than to a specific performer. Associations between users and roles are defined in the Roles and task performers list, on which basis the tasks are automatically reassigned to specific users. However, in certain cases, the performer role is not enough to accurately identify the performers. In these cases, you need to specify the main and additional addressing objects. For example, the Project manager role can have the main Department addressing object and additional Project object. Thus, addressing objects help to clarify to which project managers and in which department the task is assigned.

The map of performers to roles and addressing object is set before starting work with business processes and tasks, and it is used by the application for delivering instructions to specific performers.

The list of roles is included in the application, but can be expanded if necessary.

It is not recommended to allow the list to have roles with no performers assigned (marked in red) as this can lead to the emergence of tasks that were not completed on time. The Hung tasks report is designed to control such tasks.

The External role flag indicates that the record of performers for this role is kept in another infobase. Roles with the External role flag are used for infobase integration purposes.

The Task control manager predefined role can be used to get notifications about overdue and hung tasks for all or selected addressing objects. By default, users assigned to the Task control manager role get notifications on all such tasks assigned to any performers. If you need to track overdue or hung tasks for specific addressing objects only, you need to have the Task control manager role assigned to you for these addressing objects.

The application sends notifications to the users with the Task control manager role using the email addresses specified in contact information of these users.

The list includes the following columns:

  • Description. Full role description.
  • Short presentation. Short description of a role for your convenience. By default, Code is automatically copied into this field.
  • Code. Filled automatically.

Peculiarities of interaction with external users

  • External users can be performers of business processes (for example, partners or contact persons of partners).
  • As a rule, external users cannot use the names of specific employees of a company, thus, they can only specify the performer roles assigned to them.
  • To do that, you need to configure a list of such roles, specifying that they are designed for certain categories of external users.

Creating new role

  • Select Create and fill in the required fields.
