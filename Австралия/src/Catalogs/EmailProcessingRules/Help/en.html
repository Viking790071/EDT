This list is used to create and save conditions that govern automated sorting of incoming and outgoing mail between mail folders. The rules are set up per email account.

Email processing rules governing mail sorting between folders can be set up for each email account. These rules can be highly useful for email accounts with lots of incoming or outgoing mail.

To open the list, execute the corresponding command in navigation panel of the email account.

The list includes the following columns:

  • Condition. The name of the condition.
  • Put in folder. The folder to put the emails that match the condition.

Rules application order

  • To move conditions up or down in the list, click or .
  • The order of the conditions in the list is the order the rules are applied.
  • Top rules apply first, and the bottom rules apply last.
  • If a rule condition is fulfilled for an email, this rule determines the folder for this email and no further rules apply.

If none of the rules apply to an email, the email is moved to the default folder:

  • Inbox stores all incoming mail.
  • Outbox stores all outgoing mail that was not yet sent.
  • Junk mail stores all mail identified by mail server as junk (for more details, refer to the mail server documentation).
  • Sent items stores all sent mail.
  • Deleted items stores all mail marked for deletion.
  • Drafts stores mail to be edited before sending.

Creating rules

  • To create a processing rule, click Create in the list.

Applying rules

  • To reapply the rules to the previously sorted mail, select the rules you need and click Apply rules in the list.
  • The Apply rules command is also available in the interaction list.
