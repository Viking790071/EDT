This form is used to create and save rules for automated sorting of incoming and outgoing email by folders. The rules are configured for each Email account separately.

Configuring rules
  • In the Condition field, select a text description of the condition the rule uses to sort email.
    • This text may be generated automatically, depending on the conditions.
    • When the rule conditions are defined, select the description from the list.
    • If the autogenerated description is too long, you may type in any other text. This field is required.

  • In the Put in folder field, specify the folder where you want to put the email matching the conditions. To do this, select a folder from list. This field is required.

Entering conditions

The conditions are displayed as a list with columns containing the filter fields, values of the filter fields, and filter fields groups, if any.

  • To create a new condition, click Add new item. Multiple conditions can be used.
  • In the Field column, select an email field you want to used for filter. Specify a field and click Select.
  • In the Comparison kind column, select  a comparison operation. The list includes both common (Greater or equal) and non-standard (In list) comparison operations.
    • Comparison operation In list is used to specify the list of allowed filter values. For example, you can create a list of the partner's email accounts that are allowed to send email to you. Comparison operation Not in list is inversion of the above. For example, you can use it to put email incoming from all accounts except the accounts in the list to a specific folder.
    • Operations Contains and Does not contain are used to search for a field (name, address, or email subject) which you only know partially. For example, they can be used to identify emails incoming from contact persons of a specific partner if you know this partner's email domain. In this event, all email incoming from accounts that contain the specified sequence of characters will be put in a specific folder.
    • To find out whether a field contains a value, use the Filled and Not filled operations. For example, you can use it to put all emails with blank subject to a specific folder.
  • Then, fill in Value. Depending on the selected field and comparison type, Select a value from the list, enter a value, or create a value list (for example, a list of email addresses used by a partner).
    • Email document fields can be used as values. To enable this option, select the More – Use field as value check box. Then, Select a field from the list.
  • Select the More – Details to display additional columns in the condition list:
    • Display mode. Display mode of the value. Set to Quick access by default. Can be changed to Normal or Unavailable.
    • Presentation for Значение column.

Grouping conditions

  • To refine your filter, you can apply multiple conditions simultaneously. This can be highly useful for email accounts with lots of incoming or outgoing mail.
    • To do this, you need to apply logical AND, OR, or NOT to the selected conditions using the Group conditions command. For example, to put all email sent by a specific partner to a separate folder, you need to create a rule with conditions listing each known email address of the partner, and group these conditions using logical OR.
    • When it is known that a customer only sends email to a specific employee or from specific email accounts, you can create more precise mail filter conditions using logical OR.
    • To group multiple conditions, select a condition, execute the Group conditions command, select a Group type: AND, OR, or NOT. Drag the other conditions to this group, or create conditions directly in the group.
    • To create a group, execute the More – Add new group command.
    • To delete a group, execute the More – Ungroup command.
    • To change the order of conditions in the list, click More – Move up or More – Move down.
    • If a filtering rule includes multiple conditions without grouping, logical operation And is applied to such conditions by default.
  • Click Write and close and the created rules will be applied to all new emails of the account (when receiving emails and changing email statuses, for example, from new to draft and so on). Execute the corresponding command in email processing rules list to apply rules to old letters.
