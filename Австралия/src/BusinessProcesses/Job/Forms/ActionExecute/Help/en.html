Designed for the Job business process execution.

Opened from the tasks lists, for example, My tasks on desktop or All tasks.

Main

  • The Main tab contains Job subject, and it specifies Task and task details.

Execution history

  • The Execution history tab contains the information about the work of the performer on the task (filled automatically).
  • If the task is executed or canceled, this task cannot be changed. Please note that the Execution history field is filled by the application automatically and secured from editing.

Importance

  • Takes Normal (by default), Low, High values.
  • To select a business process from the list, or to specify that the execution of this business process might be deterred, select the matching value of this field.

Execution start date

  • In the Start date field you can enter the date and time when you started to execute the job. To enable this field, select the Task start date check box in the Administration - Organizer section. You can sort tasks in lists by this field.

Accepting for execution

  • Задача, непринятая к исполнению, в списках выделяется жирным шрифтом. Execute the mapping command of the More actions menu in order to Accept the task for execution. После этого задача исчезает из списков задач других исполнителей, назначенных на ту же роль.
  • Using the appropriate command of the More menu, you can Cancel accepting for execution.

Executing job

  • Tasks can be without execution check. In this case, the performer checks the execution.
  • Write a comment about job completion in the Job execution result field.
  • Confirm the execution (by clicking Completed).
  • After that, the execution date is filled, the task is closed and becomes secured from editing.

Non-completion

  • Write a reason for the incomplete job in the Job execution result field.
  • Click Canceled, which means the task is not completed.
  • After that, the Date of cancelling execution is filled, the task is closed and becomes secured from editing.

Forward to another performer

  • Task can be Forward to another performer using the matching button.
  • Fill in the required fields.

Create subordinate business process

  • To use subordinate business processes, select the matching check box in the Administration - Organizer section.
  • Often business processes are not created independently, but as part of the continuation of the tasks of other business processes. For example, a director instructs the managers of departments to prepare an annual plan-report. Managers give instructions to their performers according to this task. Performers begin working, create reports, send them for approval, and so on. Thus, a whole hierarchy of business processes and tasks grows from the first assignment of the director.
  • To start one business process on the basis of another, in the task of the parent business process, execute the Generate based on command, select a kind of a business process from the list, for example, Job. Fill in the required fields.

Reminder

  • In order to remember to complete the task, click Remind, fill the required fields. At the specified time, the application will display reminder on the screen.

Изменение задания

  • Using the Change job reference, you can open a job and change it (if you have the access rights). В этом случае задание снова будет направлено исполнителю.

Additional info about the task

  • To view additional information about the task, execute the More - Additional command.
