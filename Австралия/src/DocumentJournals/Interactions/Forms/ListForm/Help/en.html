Use this window for operations with interaction documents.

Nowadays, business processes involve not only standard records management but also communication between partners, their interactions. various meetings, negotiations, email correspondence. Upon their communication, business partners make important decisions, conclude agreements, draw up other documents. That is why their interactions need to be recorded.

With the application, you can schedule, register, sort, and deal with interaction results, such as the application user communication regarding the interaction subject. For example, one employee can interact with another employee (contact) regarding the sales order (subject).

With the application, you can place various interactions that are based on the same subject in a chain. In the interaction chain, one first-in-time interaction is usually specified as a subject for all interactions. At any time, in any chain interaction, you can specify another subject that is not an interaction. For example, it can be one of the application documents. After that, the subject will be changed for the whole chain automatically. For example, negotiations with a potential customer, consisting of meetings, phone calls, and correspondence (interaction chain), resulted in a sales order. In this case, the application allows you to change the subject in all interactions preceding the order to the final sales order.

For a better user experience, all application contacts are stored in the single address book.

The interaction list consists of two sections: there is usually the navigation panel on the left of the list and interaction documents on the right.

The right part of the list includes the following columns:

  • Interaction kind. Interactions in the list have the following icons:
    • The Scheduled interaction document is used to display future interactions in the application.
    • The following documents are used to display recent contacts:
      • Meeting
      • Text message
      • Phone call
      • Incoming email. You cannot add this document manually, unlike the rest of the documents. You can receive it from a sender only.
      • Outgoing email
  • Importance. Indicates whether the interaction is highly important or insignificant. Can take the following values: Low, Normal (default value), or High. It is diplayed in interaction list by special icons.
  • Participants. Contacts that had interactions.
  • Subject. A short summary of the interaction.
  • Date. The document creation date.
  • Attachment existence. You can attach different files to emails.

At the bottom of the list (depending on the settings), there can be details, a text of an interaction selected in the list, or an email body.

In the list, documents for review, emails, and text messages with set flag Under control are highlighted in bold. Unreviewed documents and emails with set flag Under control are counted automatically in each group of the list. Groups that include such documents are shown in bold.

Entering interaction documents

  • Click Create and select a document from the list.

Kinds of interaction list navigation panels

On the left of the list, there is a panel you can use to easily group interactions:

  • By subjects. The navigation panel is filled in with interaction subjects. All list interactions are grouped by subjects.
    • On subject positioning, the list shows interactions by this subject.
    • In the grouping panel, you can open subjects by which interactions are grouped (by double-clicking in the grouping panel).
    • You can drag interactions to move them among subjects. As a result, the subject changes in the interaction.
    • To quickly set a subject of interaction documents, drag emails to the subject list window.
    • You can quickly change the subject by dragging the selected interactions to the parallel window of the list of documents that may be interaction subjects.
    • Select a related check box in the context menu of the navigation panel by right-clicking to show Only active subjects or Only important subjects.
    • If you select one of the subjects in the grouping panel, click Create to enter a new interaction document by this subject.
  • By contacts. The navigation panel is filled in with contacts that had interactions.
    • On contact positioning, the list shows interactions with this contact.
    • Select the check box in the context menu of the navigation panel by right-clicking to show Only important contacts.
  • By folders. The navigation panel is filled in with user-accessible email accounts and their folders. Unlike other navigation panel kinds, it is used only for operations with emails. See details below.
  • By tabs. The navigation panel is filled in with user tabs. On tab positioning, the list will include interactions that match the filter set up in the tab. See more information about the list tabs below.
  • By properties. A separate navigation panel kind is generated for each additional attribute or information of interaction documents.
    • If you select this navigation kind, the panel is filled in with all possible values of this additional attribute or information.
    • When positioned on one of the values, the interaction list shows documents where related information has the same value.
  • By categories. The navigation panel is filled in with kinds of interaction document properties:
    • If an additional attribute or information has the Boolean type, which can be True or False, the navigation panel shows additional attribute descriptions.
      • When positioned on a property kind, only documents where the value of this property is True are displayed in the interaction list in the navigation panel.
  • If an additional attribute or information has a value other than the Boolean type, grouping that matches the additional attribute description will be available in the navigation panel.
    • With such grouping, the navigation panel shows additional attribute values and the interaction list displays interactions where additional attributes match the values selected in the navigation panel.

Switching navigation panel kinds
  • To select a navigation panel kind, use the drop-down menu located above the navigation panel.

Hiding navigation panel

  • For your convenience, you can Hide navigation panel using the appropriate command in the More actions menu.

List filters

  • To filter the list by person in charge for processing interactions, click on Person responsible and select a user from the Users list.
  • To filter the list by interaction type, click More — Interaction type and select the type. For example, Email or Phone call. The default filter value is All types.
  • To filter the list by interaction review status, click More — Status and select the status. For example, Reviewed or Deferred. The default filter value is All statuses.
    • If the filter is set to To review, the result will contain interactions that meet the following requirements: to be processed before the current date and Reviewed is cleared.
    • If the filter is set to Deferred, the result will contain interactions that meet the following requirements: Review after is filled in and Reviewed is cleared.
    • If the filter is set to Reviewed, the result will contain interactions that meet the following requirement: Reviewed is selected.
  • To filter the list by interaction period, click More — Set period.
  • Applied filters are saved.

Full-text search in texts and attachments
  • Click on Find and enter the search request.
    • The full-text search looks through texts and attachments.
    • If a text or attachment contains the request, it will be added to the result list.
    • The search result includes only texts and attachments that meet the search criteria.
    • To view the text that contains the search request, point the cursor on an interaction at the bottom of the list.
    • If no results are found, the search bar is highlighted in red.
    • To clear the list of search results, clear the search bar.
  • By default, full-text search is looking for a whole word or a phrase. Search takes into account word forms, such as number, gender, or case.
  • For advanced search, use search query language. For example, to search on part of word.
    • For more information about the search query language, go to Help, then 1C:Enterprise — User work — Query language for full-text search in data.
  • Note that the search bar treats some characters as search query language operators.

Marking document as Reviewed

  • Select documents.
  • Click Mark as Reviewed (Ctrl+R).
  • To mark a group of documents, right-click anywhere in the navigation panel and click Reviewed.
    • All filtered documents will change status to Reviewed.
    • For example, if you apply Reviewed status to the Inbox folder, Being tracked status is cleared for all emails in this folder.
  • Once you applied Reviewed status to a document, the document is deselected.

Clearing review marks 

  • Select documents.
  • Click More – Clear review mark (Alt+Shift+R). Emails change status to Being tracked.
  • Document font changes to bold.

Deferring document reviews

  • Select documents.
  • Click Defer review (Ctrl+B). Select the date and time after which the document should be reviewed.
  • To defer review for a group of interaction documents, right-click anywhere in the navigation panel and click Defer review.
    • Reviews for all filtered documents will be deferred.
    • For example, if you apply Defer review to the Inbox folder, reviews for all emails in this folder defer untill the specified date.

Viewing deferred interactions

  • To view deferred interaction documents, select Deferred status. The resulted list will contain only the interaction documents with Being tracked or Review after statuses.

Defining interaction subject

  • Select documents.
  • Click Define interaction subject (Ctrl+G). Select a document type. Then, in the list, select a document that will serve as the interaction subject.
  • To define interactions subject for a group of interaction documents, right-click anywhere in the navigation panel and click Define interaction subjects.
    • The interaction subject will apply to all filtered documents.
    • For example, if you define interaction subject to the Inbox folder, this subject is applied to all emails in this folder.

Assigning documents with responsible persons

  • Select documents.
  • Click Assign person responsible (Ctrl+M). Select a user from the list.
  • To assign a person responsible to a group of interaction documents, right-click anywhere in the navigation panel and click Assign person responsible.
    • The person responsible will be applied to all filtered documents.
    • For example, if you apply a person responsible to the Inbox folder, the user is responsible for all emails in this folder.

Adding interactions to bookmarks

  • Select documents. Right-click anywhere in the navigation panel and click Add to bookmarks.
    • A new bookmark Favorite interactions will be created.
  • To add a group of interaction documents to bookmarks, right-click anywhere in the navigation panel and click Add to bookmarks.
    • All filtered documents will be added to bookmarks. For example, if you apply Add to bookmarks to the Inbox folder, all email in this folder will be added to the new bookmark In Inbox folder.
  • To create a custom bookmark and filtere documents by custom criteria, change Navigation option to By bookmarks.
    • In the context menu of the navigation panel, click Create to create tabs. Click Create group to create tab groups.

Viewing contents of documents in the list

  • Select More – Show reading pane to view the email text or the document subject.

Grouping the list by email folders. Email operations

To make the email experience better for you, select the By folders grouping.

  • In this case, all correspondence is grouped by accounts first. Then it is grouped within each account by email folders, predefined folders, and user folders.
  • Double-click a description of an email account to open it.
  • Default predefined folders:
    • Inbox stores all incoming mail.
    • Outbox stores all outgoing mail that was not yet sent.
    • Junk mail stores all mail identified by mail server as junk (for more details, refer to the mail server documentation).
    • Sent items stores all sent mail.
    • Deleted items stores all mail marked for deletion.
    • Drafts stores mail to be edited before sending.
  • Mail is sorted between folders automatically, upon status assignment or status change, or when processing rules are applied. Click Apply processing rules in the context menu of the navigation panel to sort emails according to previously developed processing rules.
  • On folder positioning, the list shows emails of this folder.

Для работы с папками электронных писем предусмотрены команды контекстного меню панели группировки по правой кнопки мыши:

  • Move to folder. The command is available only for user folders. This requires sufficient rights. Для перемещения папок можно также использовать перетаскивание мышью. Select a folder from the list or create a new folder by using the command.
  • Create folder. To create a new folder, fill in the required fields.
  • You can drag emails to move them from one folder to another.
    • You can also click Move to folder in the context menu on the right of the list. Select a folder from the list or create a new one. Click Select to return to the list.
  • The following commands for email operations are available in folders:
    • Send email
    • Reply
    • Reply to all
    • Forward
    • Отправить и получить.
    • Переслать как вложение- воспользуйтесь этой командой контекстного меню по правой кнопке мыши, будет сформировано новое письмо, выбранное письмо будет одним из вложений.

Отправка и получение писем

Set the By folders navigation panel kind.

  • In file mode, click Send and receive mail (Ctrl+F9) and check your mailbox.

    • If there are new emails since the last mailbox check, they will be automatically included in the list. After that, a message about the number of new emails will appear.
    • With this command, only emails of the current user are sent and only emails from the accounts available to them are received.
  • In client/server mode, emails (of all users and all accounts) are sent and received using the scheduled job with an interval set by the administrator.

Replying to emails or forwarding emails from the interaction list

  • Set the By folders navigation panel kind.
  • Select a folder. In this case, the command bar of the interaction list includes appropriate commands.
  • Select an email.
  • Click the related button:
    • Reply. Creates an outgoing email addressed to the sender of the original email. The outgoing email is included in the interaction chain.
    • Reply to all. Creates an outgoing email addressed both to the sender and all recipients of the original email (specified in the To and Cc fields).
    • Forward. Creates an outgoing email without a recipient. The outgoing email includes the text and attachments of the original email.
    • Forward as an attachment. Right-click the command in the context menu to generate an outgoing email, which includes the selected incoming or sent emails as an attachment.

Personal settings

  • Click More actions – Personal settings to go to setup of a personal email account.
