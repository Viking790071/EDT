Customizing account usage in the default email client
  • Settings are made on the Main tab using the In default email client radio button:
    • Select the Receive, save, and send emails from this account value if you plan to keep correspondence with this account.
    • If you are not going to use the account in the internal email client (for example, this account is used for exchanges with other applications), select the Do not use value.

Including a username specified in the account in the email sender presentation

  • Select the Include username in presentation check box on the Main tab.

Purpose of the Delete emails after sending check box

  • Setting is made on the Main tab.
  • If there is no need to store outgoing emails successfully sent from this account, this check box must be enabled.
  • In this scenario, after sending emails, they will be deleted.

Collective account

  • For a collective account, one user can be responsible for folders, and another default user (for example, the application administrator) can be responsible for emails processing.
    • Select Person responsible for email processing from the user list. This field is required.
    • Select Person responsible for folder management from the user list. This field is required.

Example of using a collective account

  • A large company has a large number of clients throughout the country.
  • The company has a single postal address to which clients send their claims.
  • The content and order of the folders for this account is determined by the service manager, who is responsible for processing claims.
  • The person responsible for the initial of parsing emails is another employee. They process incoming correspondence and assigns letters to responsible persons, who will then be responsible for processing claims in emails.

Purpose of the Place emails to base email folder check box

  • Setting is made on the Email processing tab.
  • If this check box is selected, the processing rules will not be applied to emails that are a reply or forwarded email.
  • Such emails will always be placed in the same folder as the base email.

Use another email client to process emails (when receiving emails using the IMAP protocol)

  • If you receive emails using the IMAP protocol, processing of incoming emails can be carried out in another email client.
  • Using the radio button on the Email processing tab, you can specify Where emails are processed:
    • In another email client. In this case, the application email client is rarely used, for example, only for sending printed forms, or other options of sending emails using infobase data are used.
      • The main operations with emails, including email parsing, are carried out in another email client.
      • In this case, the Review flag is synchronized in the application with a read flag on the email server.
      • Synchronization is executed unilaterally. The flags of reading from the server are received and set on the email client of the application. Changes of the read flag are not transferred from the client to the server.
      • When receiving emails, the Reviewed flag always maps the status of the email on the email server.
    • In this email client. Parsing correspondence is carried out in the application. In this case, all incoming letters loaded to the application are not reviewed.

Set up an email signature
  • You can create a signature of two kinds:
    • for new messages using the Signature for a new email tab.
    • for messages when replying or forwarding using the Signature on replying tab.
  • Select the Enable check box for the signature that you need.
  • Select a signature format from the list:
    • Plain text.
    • HTML. Creation of a signature in an HTML format allows you to format text and insert pictures in it.
  • In order to place signatures, select the Enable signature for new emails, Enable signature on replying or forwarding check boxes.
  • Write a signature to the mapping field. If you select the HTML format, you can format the text in various ways, insert images and hyperlinks. You can format text using navigation panel buttons and also using the More menu commands.
  • It is necessary to remember that, when you change a previously entered signature from HTML to Plain Text, some of the information (for example, pictures, hyperlinks and other elements that exist only in the HTML format) is permanently deleted. The application displays a warning about this. Click Yes to confirm.
  • Also, signatures can be set up using the More - Personal settings command from the interaction list.

Important. You can create signatures in HTML format if it is provided by the application settings. You can change this setting in the Administration - Organizer section by selecting the Outgoing emails in HTML format check box.
