This is the list of application objects that can be used for role-based addressing purposes together with the performer roles.

Opened by the matching command of the Business processes and tasks section.

The list includes the following columns:

  • Description. Full description of an address object.
  • Value type. A value type of an addressing object.

Depending on the specifics of business processes, tasks can be addressed to a performer or to a group of performers both personally (personal addressing) and, for some business processes, using business process roles (role addressing).

In case of role addressing, assignment is not issued to the performer, but to a certain role of the business process (for example, Room supervisor). Specifying, main, and additional addressing objects can be used. For example, the Project manager role can have the main Department addressing object and additional Project object. Thus, addressing objects help to clarify to which project managers and in which department the task is assigned. The map of performers to roles and addressing object is set before starting work with business processes and tasks, and it is used by the application for delivering instructions to specific performers.

The All addressing objects predefined item covers all addressing objects available in the application.
