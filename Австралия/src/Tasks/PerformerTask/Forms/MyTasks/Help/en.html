Use this form to work with tasks. The list contains tasks addressed to the current user.

Открывается по команде Мои задачи в разделе Бизнес-процессы и задачи.

Tasks not assigned for execution are marked in bold. Overdue tasks are highlighted in red.

The list includes the following columns:

  • Importance. Takes Normal (default), Low, High values. Use this field to specify that execution of a business process can be postponed or, to the contrary, should be accelerated.
  • Number. Assigned automatically. Completed tasks are marked with icon.
  • Task. Full description of the task.
  • Start date of task execution. Filled in by the assignee. If not filled, the current date is selected automatically. To have this field editable, select the Date of task start check box in the Administration – Organizer section.
  • Due date. Date and time by which Job is to be completed. To have time displayed for the dates in this list, select the Date and time in task deadlines check box in the Administration – Organizer section.
  • Written. The date and time of task creation.

Grouping

  • If there are a lot of tasks, for your convenience, you can group the list using the matching button Group by:
    • due date
    • importance
    • route point
    • author
    • task subject
  • To cancel grouping, click Group by – No grouping.

Filter by period

  • Execute the context menu command by right-clicking, or find the command in the More - Set date interval menu.

Viewing a task

  • To view tasks, double-click the required string. You can also execute the Open task command of the context menu by right-clicking, or find the command in the More menu.

Viewing a subject

  • Tasks can be created based on application objects. To view the task subject, execute the Open task subject command of the context menu by right-clicking, or find the command in the More menu.

Viewing a business process

  • Each task is a step of a business process. To view information about the business process as a whole, execute the context menu command by right-clicking, or find the command in the More - Open business process menu.

Accepting for execution

  • Tasks not yet accepted are marked in bold.
  • Use the matching button to accept a task For execution. In the case of role addressing, the task accepted for execution disappears from the tasks lists of other performers who are assigned to the same role.
  • In order to refuse to execute the previously accepted task, you must use the context menu command by right-clicking, or find the command in the More - Cancel accepting the task for execution menu.

Execution

  • Use the context menu command by right-clicking, or find the command in the More - Completed menu to mark the task as completed.
  • Completed tasks are mark with the icon in the list.

Forward to another performer

  • Click Forward, fill data about another performer.

Подчиненные бизнес-процессы

  • A new business process, for example, Job, can be created on the basis of this tasks by selecting Create on basis. This requires access rights.

Напоминание

  • Для того чтобы не забыть выполнить задачу, можно ввести Напоминание, текст которого будет выведен в указанное время.
  • Click , select an action, for example, Remind, fill the required fields.
