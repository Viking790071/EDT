With this form you can add contacts to interaction documents.

All interaction documents contain Contact field, but the purpose of contacts depends on the document type.

  • For Meeting and Scheduled interaction, contacts mean the people who are supposed to participate in the event.
  • For Text message, contacts mean the group of people who will receive the message.
  • For Phone call, contacts mean the call recipient.

The Select contact form opens when you start filling in the Contacts field in an interaction document.

The form contains the following tabs:

  • Search contacts searches contacts in the application by the following attributes:
      • By Email.
      • Email server domain.
      • Phone number.
      • Keywords in the contact description.
        • It is a full-text search that looks through all contact attributes.
        • In the search window bottom, you can see the Refresh button, which is active when the search index is outdated.
      • Contact description leading characters (Begins with).
    • Select a search option, enter a search request, and click Find. The search result will contain the contacts that meet the conditions.
  • Users. Select a contact from the application user list.
    • The tabs can contain another lists. For example, a list of partners.
    • For a faster browsing, use Find. The result list supports all commands and actions.
  • Contacts by subject. The tab is active if the interaction document Subject is filled in. The tab contains the list of contacts related to the subject. The list includes contacts that are mentioned in the emails and interaction documents with this subject.

Adding contacts to interaction documents

  • Double-click on a contact to add it to the interaction document, or select a contact and click Select.
  • To add multiple contacts, select all of them and click Select.
