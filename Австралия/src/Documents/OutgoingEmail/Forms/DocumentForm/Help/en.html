Default email account
  • The application automatically populates the From field (sender email address) with the following:
    • Basis email account if this email is generated on the basis of other email. For example, as a reply to incoming email.
    • The first of available user accounts if an email is not generated on the basis of other email.
  • Учетную запись, с которой предполагается отправка письма, можно изменить. To fill sender address, choose an email account from dropdown list.

Адресаты письма

  • Email addressees can be specified in three lists:
    • To. Main email addressees that take part in communication.
    • Cc. Additional email addressees. This field usually contains those users who do not directly participate in communication but must be informed about its content.
    • Hidden. Addressees specified in this field will receive an email but other email addressees will not know about it.
  • If application has required contacts, you can specify addressee using two methods:
    • Input by string. Start to input addressee name or his email address and the application will offer you an addressee by found matches. This addressee will be inserted into email. You can enter both contact name and his email address. You can enter several addresses, separated by semicolons.
    • Using addressee pickup. Click Select in the field with addressee. Address book will open, using which you can pick addressees.
  • If there are no contacts in the application, you can fill in fields with email addressees manually. After an email is sent, the Specify contacts command becomes available, using which you can call Address book.

Fill in the following required fields:
  • Subject. Brief details of what the email is about.
  • Email. Email text itself, placed on the associated tab.
  • Comment. An arbitrary text comment, additional information about an email. Располагается на специальной вкладке.
  • List of attachments. You can attach different files to an email using navigation panel commands.
  • Предмет - то, чему посвящено взаимодействие, например, претензии клиента. Subject is adopted from an application document or interaction document. If the interaction subject is not specified, it is automatically copied from the current document, meaning that interaction chain starts with this interaction document. When you click Interactions from the document, the new document inherits the interaction subject and main fields from the parent document.
    • To fill in this field, click Select, choose data type, and select a document from the list.
  • Person responsible. Select a person responsible for the interaction from the users list. By default, for the most of interaction documents, the current user is selected as a person responsible.
  • Under control. Indicates that this email will require attention in the future. It is displayed in bold in the interaction list. It is recommended to use the check box when sharing interaction documents with your team.
  • Review later. It must be filled in if you plan to postpone control of this letter until a certain date. Select one of the following values from the list: In 15 min, In 30 min, In 1 hour, In 3 hours, or Tomorrow. To set a custom date and time, click .
  • Importance. Indicates whether the interaction is highly important or insignificant. Possible values are Low, Normal (default value), and High. It is diplayed in interaction list by special icons.
  • Created. Date and time the document is created. The field is filled automatically with the date and time when the document was written. The field cannot be edited.
  • Author. A user who created the document. The field is populated automatically. The field cannot be edited.

You can set the application settings to show or hide the Under control and Review later attributes. You can change the settings in Administration - Organizer.

Editing email in HTML format
  • You can edit email in HTML format if it is provided by the application settings. You can change this setting in the Administration - Organizer section by editing the Outgoing emails in HTML format check box value.
  • The default email format is defined according to the format of the previous email written by the user.
  • You can change editing format using the dropdown menu when entering the letter.
    • There are two email editing formats available:
      • Plain text. When switching email into plain text all appearance items, pictures and other inserted items will be lost. The application displays a warning about this. To confirm the action click Yes, to cancel click No.
      • HTML. You can format text and insert pictures and references to it using standard commands.
        • Standard formatting buttons are above the email text. Also you can find text formatting commands in the More menu.
  • After saving the entered email, you will not e able to edit its format.

Adding attachments

  • List of attachments is in the right part of the email above the Subject field.
  • Use the Add attachment command of attachment list context menu.

Add email as an attachment

  • Execute the Add email command of attachment list context menu. Select an email from the list.

Attachment operations.

If an email has attachments, then in the list of attachments by right clicking the context menu you can:
  • Open attachment. Open file using the mapping external application.
  • Delete attachment. You can also delete an attachment file by pressing Del.
  • Свойства вложения - просмотр карточки присоединенного файла. You can also sign an attachment with a digital signature or encrypt it using the navigation panel commands in the card.

Включение текста исходного письма при ответе или пересылке
  • Execute the More - Email parameters command and select the Include original email body check box.
  • By default this value is populated from personal settings that can be changed from the list of interactions.

A delivery or read receipt of an email
  • Execute the More - Email parameters command and select the Request delivery receipt and Request read receipt check boxes. By default this values are populated from personal settings that can be changed from the list of interactions.
  • Other mail parameters and service information are displayed here.

Email encoding choice
  • Execute the More - Change encoding command and select the necessary encoding from the list.

The Specify contacts link
  • The application allows you to specify list items as interaction participants. For example, clients or suppliers. This allows you to further get information about all interactions with the client. If at least one addressee will not be mapped to contact when creating an email, it will display the Specify contacts reference. The reference opens Address book, where you can map addressees to contacts.
  • If all contacts are mapped, then this reference is not displayed. You can map contacts by the More - Specify contacts command.

Changing email folder
  • Execute the More - Email parameters command and select other folder in the opened window.

Sending email
  • To send an email (in file mode) click Send. All emails by all users and all accounts are sent and received in client/server mode as a scheduled job.

Create interaction documents on the basis of an outgoing email
  • Click Interaction and choose one of the following:
    • Schedule interaction
    • Schedule meeting
    • Send SMS
    • Make call

Viewing interactions generated from email

  • To view the list of interactions generated from this outgoing email, click Interactions on the navigation panel.

Printing an email
  • Click Print.
  • In the Print window, specify printer settings, such as number of copies, paper size, and more.

Outgoing email statuses

Outgoing emails receive the following statuses:

  • If an email has not been sent yet, it has the Draft status. You can edit it and save changes. You can Send email by clicking the corresponding button (for file mode) and it will receive other status.
  • If an email is in the process of sending, it receives the Outgoing status. After an attempt to send an email its status changes and button description changes to Send now. You cannot edit such emails.
  • After sending an outgoing email its status changes to Sent. You cannot edit such emails either.
  • Depending on the status, an email is placed in one or another folder. To place emails into folders you can also set up email processing rules.

Insert references to infobase objects into an email

  • Using the corresponding command of More menu, you can Insert external reference to infobase object. The address, by which an external user can access infobase, is populated automatically. To get an internal reference to object, open the necessary object, press Ctrl+F11 and insert the reference to corresponding field. Click Insert and email will get the required reference.

Save email to the computer

  • Execute the More - Save email to disk command.
  • Specify a path to save the file on the computer. The email is saved in the EML format.
