This document is designed to send text messages to one or more recipients, bulk messaging, and display the fact that messages are sent.

The document allows you to create, edit, and send text messages, in addition to recording and storing. It is possible to send several messages with the same text to different recipients using one document. The state of the document and the state of a separate message to each of the different recipients within the document are tracking.

Fill in the following required fields:
  • Subject of the interaction, for example, customer claims. The subject can be program documents, as well as interaction documents. If the interaction subject is not defined, the current document serves as a subject. It means that such document begins the interactions thread.
  • To fill in this field, click Select, choose data type, and select a document from the list.
  • Person responsible. Select a person responsible for the interaction from the users list. By default, for the most of interaction documents, the current user is selected as a person responsible.
  • Under control. Shows whether the text message requires attention after sending. For example, if it is necessary to check that the message is delivered to the recipient. In the interaction list such text message is marked in bold.
  • Review later. If you plan to defer control of this message until a certain date, fill in this field. Select one of the following values from the list: In 15 min, In 30 min, In 1 hour, In 3 hours, or Tomorrow. To set a custom date and time, click . If the Under control check box is selected, it is available.
  • Convert Cyrillic characters into the Latin ones. Shows whether the message must be sent in transliteration.
  • Message. The text that will be sent to all specified recipients. See the associated tab. The program counts the letters of the entered text and indicates the number of remaining characters to the end of the message according to the providers rules. If the size of a single message is exceeded, the text is automatically divided into several separate messages.
  • Comment. An arbitrary text comment. See the associated tab.
  • Created. Date and time the document is created. The field is filled automatically with the date and time when the document was written. The field is not editable.
  • Author. A user who created the document. The field is populated automatically. The field is not editable.

The Under control and Review later fields are available for filling, if it is provided by the application settings. You can change the settings in Administration - Organizer.

Recipients
  • You can enter recipient information in the Recipients list.
  • To specify one or more contacts, click Add.
  • Contacts are required.
  • If the recipient is already registered in the infobase, specify information in the Contact field. Select contact from the Select contact list. After that, the remaining fields are filled in automatically.
  • If you do not plan to store information about the message recipient, it is enough to specify Contact presentation (full description) and Phone number. In this case, the Not all contacts selected message is displayed on the right side of the document.
  • The phone number must be specified in format +xx (xxx) xxxxxxx. The field is required.
  • If this is a new recipient, click Create contact to register it directly from the document window. Select the matching list and enter the new recipient data.

Send messages

  • To send text messages to all listed contacts, click Send.
  • After clicking this button:
    • In file mode, user sends the message.
    • In client server mode, the user initiates the sending and message state is changed to Outgoing. Message is sending in the background by the "Send text messages" scheduled job.

Checking the current text message state

  • The message delivery state is checked automatically according to the schedule configured by the application administrator.
  • You can also check the delivery state of document messages from the document window by using the Check delivery state command.
  • The state of text messages is displayed at the bottom of the document and in the table of recipients in the Message state column for each contact separately.
  • The state of message to a specific recipient is displayed in the Recipients tabular section. Может принимать одно из следующих значений:
    • Draft. The message has not been sent yet and can be edited.
    • Исходящее - пользователь уже отправил сообщение, но оно еще не передавалось провайдеру.
    • Cannot sent it to provider. The application attempted to send a message to the provider, but failed. For example, if there are technical problems on the provider side.
    • Not sent by provider. The message was successfully sent to the provider, but has not yet been processed.
    • Being sent by provider. The message is currently being sent by the provider.
    • Sent by provider. The message is sent by the provider, the delivery state is currently unknown.
    • Delivered. The message is successfully delivered to the recipient.
    • Not delivered. Cannot deliver the message to the recipient. There will be no further attempts to deliver the message.
    • Unidentified by provider. The provider did not identify the message.
    • Error occurred while receiving status from provider. Cannot contact the provider when attempting to receive message state.
  • The document state is displayed in the lower left part of the document window. Может принимать одно из следующих значений.
    • Draft. The message was not sent and is editable.
    • Исходящее - пользователь уже отправил сообщение, но оно еще не передавалось провайдеру.
    • DeliveryInProgress. All messages are sent to the provider and at least for one of them delivery to the recipient in progress.
    • Delivered. All messages are successfully delivered to the recipients.
    • Partially delivered. Delivery of messages is stopped, messages are partially delivered to the recipients.
    • Not delivered.
  • The delivery state is checking by the "Update SMS delivery statuses" scheduled job. To check it manually, use the More - Check delivery state command on the Recipients tab (the command becomes active after sending messages).
